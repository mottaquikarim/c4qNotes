Memory

Definition

Think of memory as a series of folders in a very long filing cabinet.

Each file in our "cabinet" contains two pieces of important information.

  1. the address: in our example the label of the file
  2. the value: in our example, the "stuff" that is inside the file.

Memory Address

Each "file" in our "cabinet" has a numerical position. The computer can use this numerical position to look up the information that you need.

Memory Value

The memory value can be anything, some text, a number, a true or false flag, etc. This is also stored as a number, but our program can interpret this value and render for us text, numbers, etc.

The question then becomes: how are these numbers, the address and the value, stored in memory?

Bits and Bytes

Everything that we store in a computer is represented as a bit or a byte. Let's explore how these quantities work before we move on with our memory discussion.

BITS

  • The bit is the smallest unit of storage in a computer.
  • The familiar 0s and 1s you see all over movies and stuff when people talk about computers are bits.
  • A bit can only take the value of 0 or 1.
  • One bit by itself isn't super useful -- but we can combine a series of bits and interpret the combinations as a way to do useful things.

BYTES

  • A byte is a series of 8 bits.
  • We've heard the term byte a lot -- images are kilobytes, our iphone's storage is usually measured in gigabytes, etc.

Ok great, but what is a Byte, really?

It's a number. Bytes are numbers. Let's look at this real quick.

If one bit contains 0 or 1, two bits can contain:

00, 01, 10, 11

By itself, a series of 1s and 0s are pretty lame. BUT. We can encode them like so.

2 1 Math Value
0 0 (2)(0) + (1)(0) 0
0 1 (2)(0) + (1)(1) 1
1 0 (2)(1) + (1)(0) 2
1 1 (2)(1) + (1)(1) 3

And BOOM. We have numbers! Now 2 bits only gives us 4 possible numbers (0-3).

If we were to tack on ONE more bit, we would have the following:

Number of Bits Combinations Values
1 bit 0, 1 0, 1
2 bits 00, 01, 10, 11 0, 1, 2, 3
3 bits 000, 001, 010, 011,100, 101, 110, 111 0,1,2,3,4,5,6,7

Variables

We can't. This is why variables exist.

Definition

A variable is a location in memory that contains a quantity of information called value and a symbolic name called identifier.

Basically, we associated the memory address 0x11cedf with a symbolic name, like superAwesomeVariable. This abstracts away the need for us to remember or in most cases, care about the memory address. We can let the our program's interpreter deal with that.

Tying this back to memory and bytes, there are different types of variables.

A character represents a single letter such as 'a' or 'b'. It is defined as one byte.

A boolean represents true or false. It is also usually one byte.

An integer represents a number. It is usually 4 bytes. (FYI: 4 bytes = 32 bits, so the max integer would be 2^32 - 1).

And so on.

Variable Types

Most languages have a concept of primitive types and composite types.

Primitive Type

Primitive types are basic building blocks for that language, kind of like bits are the basic building blocks for memory. You can use primitives to build composite type variables, which are made up of primitives.

In javascript, there are 5 primitives:

  • undefined
  • null
  • boolean
  • string
  • number

Composite Type

Composite types are variable types that can be made up of the primitive types.

In javascript there are 3 composites:

  • objects
  • arrays
  • functions

Control Flow

Definition

Control flow is the order in which lines of code are executed in a function.

A very important aspect of programming is the ability to alter the way lines of code are exectuted.

A conditional statement is one way we can alter the way lines of code are executed.

How conditional statements work

Conditional statements evaluate to true or false and based on this evaluation, allow or block certain commands from executing.

For example:

if ( true ) {
// we write lines of code here
// these lines will always execute
}

if ( false ) {
// we write lines of code here
// these lines will never execute
}

Control Flow Operators

We can enrich our conditional statements but applying the following comparison operators into the syntax.

  1. ===, equality operator
  2. !==, inequality operator
  3. >, greater than operator
  4. >=, greater than or equal to operator
  5. <, less than operator
  6. <=, less than or equal to operator

For example,

var foobar = 'baz';
if ( foobar === 'baz' ) {
// will only execute if 'baz' is set
}

We also have logical operators:

  1. ||, the logical OR operator
  2. &&, the logical AND operator
  3. !, the logical NOT operator

For example,

var num = 0;
if ( !num ) {
// will only execute if num is 0, since !0 gives us true
}

Compound conditional statements

We can also chain our logic if necessary. We will demonstrate in examples:

if ( SOMETHING) {
// only if SOMETHING is true
}
else {
// all other cases
}

if ( SOMETHING === 1 ) {
// if SOMETHING is 1
}
else if ( SOMETHING_ELSE === 2 ) {
// if SOMETHING is 2
}
else {
// all other cases
}
